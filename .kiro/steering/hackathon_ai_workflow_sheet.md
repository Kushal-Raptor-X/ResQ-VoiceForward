# 🚀 24-Hour Hackathon AI Workflow Sheet (Your Stack)

---

## 🧠 TOOL STACK (LOCKED)

- 💡 Ideation: ChatGPT / Claude
- ⚙️ Setup: Codex (UI) 
- 🧠 Build: Antigravity / Kiro IDE
- 🤖 Iteration: Jules (limited use)
- 🔧 Debug/Polish: VS Code + Copilot
- 🗄️ Backend: Local (Node + Express + SQLite/Postgres)

---

# ⏳ PHASE 0: PRE-HACKATHON (OPTIONAL BUT ELITE)

### 🎯 Goal: Reduce thinking during hackathon

- Prepare 2–3 idea directions
- Keep prompt templates ready
- Install all tools beforehand
- Test local DB setup once

---

# ⏱️ PHASE 1: IDEATION (0–1.5 hrs)

### 💡 Tool: ChatGPT / Claude

**Steps:**
- Generate 5 ideas
- Select 1 based on:
  - Simplicity
  - Demo impact
  - Offline feasibility

### 📄 Output:
- Clear idea
- Rough understanding of features

---

# 🧩 PHASE 2: MINI SPEC (1.5–2 hrs)

### 💡 Tool: ChatGPT

**Steps:**
- Convert idea → structured spec
- Limit features to 3–5

### 📄 Output:
- Features
- Tech stack
- Architecture
- Demo flow

---

# ⚙️ PHASE 3: SETUP (2–3 hrs)

### 💻 Tool: Codex (UI)

**Tasks:**
- Setup frontend (Next.js)
- Setup backend (Express)
- Setup DB (SQLite/Postgres local)
- Setup basic auth (JWT)

⚠️ Keep it minimal (no over-engineering)

### ✅ Verification:
- App runs
- API works
- DB connects

---

# 🧠 PHASE 4: CORE BUILD (3–14 hrs)

### 🧠 Tool: Antigravity / Kiro

**Tasks:**
- Build core features
- UI + API integration
- Iterate fast

### 🔁 Loop:
Build → Test → Fix → Repeat

---

# 🤖 PHASE 5: JULES (MIDWAY ONLY) (14–18 hrs)

### 🤖 Tool: Jules

**Use Cases:**
- Fix complex bugs
- Refactor messy code
- Improve backend logic

### ⚠️ Rules:
- Use max 2–3 times
- Always constrain prompts
- Review changes carefully

---

# ⚡ PHASE 6: FINAL POLISH (18–22 hrs)

### 🔧 Tool: VS Code + Copilot

**Tasks:**
- Fix bugs
- Improve UI
- Optimize demo flow

⚠️ DO NOT USE JULES HERE

---

# 🧪 PHASE 7: DEMO PREP (22–24 hrs)

### 🎤 Focus:
- Smooth demo flow
- No crashes
- Clear storytelling

### 🧪 Checklist:
- App runs offline
- No dependency issues
- Data is preloaded

---

```
Idea 💡
  ↓

  ↓
```

- ✅ Keep scope SMALL
- ✅ Prioritize DEMO over code quality
- ✅ Avoid internet dependency
- ✅ Use Jules sparingly
- ✅ Always verify before moving forward
